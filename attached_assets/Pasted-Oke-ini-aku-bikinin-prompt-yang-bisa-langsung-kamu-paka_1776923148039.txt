🔧 PROMPT PERBAIKAN APLIKASI (React Native / Expo)

1. 🧠 BUG: Keyboard menutupi input (Catatan / Form)

Problem:
Saat user mengetik di halaman “Catatan Baru”, keyboard menutupi field input sehingga UX buruk.

Expected:

* Input tetap terlihat saat keyboard muncul
* Layout ikut naik dengan smooth
* Support iOS & Android

Fix Implementation:

* Gunakan KeyboardAvoidingView
* Tambahkan behavior="padding" untuk iOS, height untuk Android
* Wrap seluruh screen atau bottom sheet
* Gunakan ScrollView + keyboardShouldPersistTaps="handled"

Contoh:

import { KeyboardAvoidingView, Platform, ScrollView } from 'react-native';
<KeyboardAvoidingView
  style={{ flex: 1 }}
  behavior={Platform.OS === 'ios' ? 'padding' : 'height'}
>
  <ScrollView keyboardShouldPersistTaps="handled">
    {/* Form input */}
  </ScrollView>
</KeyboardAvoidingView>

Tambahan kalau pakai bottom sheet:

* Gunakan react-native-keyboard-aware-scroll-view

⸻

2. 🎨 IMPROVEMENT: Feature List jangan vertical (boros layar)

Problem:
Menu seperti:

* AI Keys
* Pack Manager
* Image Manager
* dll

ditampilkan vertical list, boros space

Expected:

* Grid / flex wrap
* Icon + label
* Responsif (2–3 kolom)
* Scroll tetap smooth

Fix Implementation:
Gunakan FlatList dengan numColumns

<FlatList
  data={features}
  numColumns={3}
  keyExtractor={(item) => item.id}
  renderItem={({ item }) => (
    <TouchableOpacity style={styles.card}>
      <Icon name={item.icon} />
      <Text>{item.title}</Text>
    </TouchableOpacity>
  )}
/>
card: {
  flex: 1,
  margin: 8,
  padding: 16,
  borderRadius: 12,
  alignItems: 'center',
}

Optional:

* Gunakan flexWrap: 'wrap' kalau bukan FlatList
* Tambahkan animasi kecil (scale on press)

⸻

3. 🚨 BUG: Import Anki error “file terlalu besar”

Problem:
Import file .apkg / JSON gagal karena limit size

Goal kamu:
Semua file disimpan LOCAL (offline) tanpa limit size

Root Cause kemungkinan:

* Limit dari state memory
* Limit dari JSON parsing
* Limit dari Expo FileSystem handling
* Atau upload ke server (harusnya tidak)

Expected:

* Import file besar tetap jalan (10MB, 50MB, bahkan 100MB)
* Tidak crash
* Parsing bertahap

Fix Implementation:

✅ Gunakan:

* expo-file-system
* expo-document-picker

✅ Jangan:

* Simpan file besar di useState
* Parse JSON langsung sekali

⸻

Contoh flow:

import * as DocumentPicker from 'expo-document-picker';
import * as FileSystem from 'expo-file-system';
const pickFile = async () => {
  const res = await DocumentPicker.getDocumentAsync({
    type: '*/*',
    copyToCacheDirectory: true,
  });
  if (res.assets) {
    const file = res.assets[0];
    // simpan path saja
    const fileUri = file.uri;
    // baca sebagai string (hati-hati size besar)
    const content = await FileSystem.readAsStringAsync(fileUri);
    // parsing bertahap kalau besar
  }
};

⸻

Advanced Fix (RECOMMENDED):

* Gunakan SQLite (expo-sqlite)
* Simpan data per chunk (bukan 1 JSON besar)
* Parse .apkg pakai unzip (karena apkg = zip)
